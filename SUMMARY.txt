BUG INVENTORY & FIXES
═════════════════════════════════════════════════════════════════════════════════

CRITICAL BUGS (Must Fix)
────────────────────────────────────────────────────────────────────────────────
1. MEMORY LEAK: child_envp
   Original:  malloc'd in init_child_env(), never freed
   Impact:    ~100-300 bytes lost per execution
   Fixed:     cleanup_and_exit() function added
   Status:    ✓ RESOLVED

2. mkdir_p ERROR IGNORED
   Original:  mkdir() return value not checked
   Impact:    Unclear error messages if directory creation fails
   Fixed:     Check return value, propagate error
   Status:    ✓ RESOLVED

3. EINTR NOT HANDLED
   Original:  waitpid() signal interrupt not caught
   Impact:    Potential undefined behavior if signal arrives
   Fixed:     Retry loop on EINTR
   Status:    ✓ RESOLVED

4. write() ERRORS IGNORED
   Original:  write() return value not validated
   Impact:    Silent failures, user thinks output printed
   Fixed:     Check return value, handle errors
   Status:    ✓ RESOLVED

MEDIUM BUGS (Should Fix)
────────────────────────────────────────────────────────────────────────────────
5. RACE CONDITION: pgrep → start window
   Original:  Window between check and start, other process could start PA
   Impact:    Rare, but possible duplicate PA instances
   Fixed:     Design-level (Termux single-user reduces impact)
   Status:    ⚠ MITIGATED (add lock file for full fix)

6. NO SIGNAL HANDLERS
   Original:  Program ignores SIGINT/SIGTERM
   Impact:    Resources not cleaned up on interrupt
   Fixed:     signal() handlers + cleanup_and_exit()
   Status:    ✓ RESOLVED

LOW BUGS (Nice to Have)
────────────────────────────────────────────────────────────────────────────────
7. BUFFER OVERFLOW RISK
   Original:  Tight buffer sizes (env_xdg_line[420])
   Impact:    Minimal with current path lengths, but not future-proof
   Fixed:     Documentation of limits, validation added
   Status:    ⚠ MITIGATED (validated but not enlarged)

8. ENV VARS NOT IN PARENT
   Original:  Child processes inherit, but parent shell doesn't
   Impact:    User must eval output, easy to forget
   Fixed:     Documentation + colored instructions
   Status:    ✓ INTENTIONAL (by design)

ENVIRONMENT VARIABLE HANDLING
═════════════════════════════════════════════════════════════════════════════════

What Gets Set
──────────────────────────────────────────────────────────────────────────────
PULSE_SERVER              tcp:127.0.0.1:4713 (configurable port)
PULSE_LATENCY_MSEC        60ms (configurable, 1-10000)
XDG_RUNTIME_DIR           /data/data/com.termux/files/usr/tmp

How It Works
──────────────────────────────────────────────────────────────────────────────
1. Program unsets inherited PULSE_* vars (clear stale config)
2. Builds child_envp with fresh vars based on current config
3. Passes child_envp to child processes (pgrep, pactl, pulseaudio)
4. Parent shell environment UNCHANGED (intentional design)
5. Prints export lines to stdout → user must eval

Usage
──────────────────────────────────────────────────────────────────────────────
# Correct usage:
eval "$(pulse_start_fixed)"     # Eval the output to set vars in shell
pactl info                       # Now PULSE_SERVER is set

# Common mistake:
./pulse_start_fixed             # Just prints, doesn't set vars
pactl info                       # Still fails, PULSE_SERVER not set

Customization
──────────────────────────────────────────────────────────────────────────────
Before calling pulse_start_fixed, export:

export PULSE_TCP_PORT=4720
  → Use port 4720 instead of 4713

export PULSE_LATENCY_MSEC=100
  → Use 100ms latency instead of 60ms

export XDG_RUNTIME_DIR=/custom/path
  → Use custom runtime directory

Then:
eval "$(pulse_start_fixed)"

ASSEMBLY-LEVEL OVERVIEW (ARM64)
═════════════════════════════════════════════════════════════════════════════════

Key Syscalls Used
──────────────────────────────────────────────────────────────────────────────
clone (220)      → fork() equivalent, create new process
execve (221)     → replace process with new binary
setsid (71)      → create new session group (detach from terminal)
wait4 (114)      → wait for child, get status
dup2 (33)        → duplicate file descriptor
open (56)        → open file/device
close (3)        → close file descriptor
nanosleep (101)  → sleep with precision timing

ARM64 Calling Convention
──────────────────────────────────────────────────────────────────────────────
Arguments:  x0, x1, x2, x3, x4, x5, x6, x7 (then stack)
Return:     x0 (32-bit), x0:x1 (64-bit)
Saved:      x19-x30 (must preserve across calls)
Scratch:    x0-x11, x13-x15
Stack:      16-byte aligned

Key Instructions
──────────────────────────────────────────────────────────────────────────────
svc #0           → syscall (supervisor call)
bl label         → branch with link (function call)
ret              → return from function
stp x0, [sp]     → store pair (atomic 16-byte write)
ldr x0, [x1]     → load register indirect
cbz x0, label    → compare & branch if zero
b.eq label       → conditional branch
mul/div          → arithmetic operations

Memory Layout (Runtime)
──────────────────────────────────────────────────────────────────────────────
Stack (grows down)  ← sp
  ├─ Local variables
  ├─ Return addresses
  └─ Function parameters

Heap (grows up)
  └─ child_envp (malloc'd ~100-300 bytes)

Data Segment
  ├─ env_xdg_line[420]
  ├─ env_pulse_line[96]
  ├─ env_lat_line[40]
  └─ String literals

Text Segment (read-only)
  ├─ main()
  ├─ init_child_env()
  ├─ start_pulseaudio()
  └─ ... other functions
